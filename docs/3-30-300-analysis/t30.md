# T30 Module Documentation

## Overview

The T30 module provides functions for calculating canopy cover (T30) using canopy height model (CHM) data from the Vegetation Object Model (VOM). This module is part of the 3-30-300 analysis framework for England using big spatial data.

## Usage Examples

### Basic Workflow

The typical workflow for T30 analysis involves:

1. **Binarising CHM tiles** - Converting canopy height model data to binary format based on height thresholds
2. **Calculating canopy cover** - Computing the percentage of canopy cover for each geometry
3. **Processing geographic areas** - Running the complete analysis for specific geographic codes

### Example: Processing a Geographic Area

```python
from src.t30 import process_geo_code
from pyspark.sql.session import SparkSession

# Initialize Spark session
sedona = SparkSession.builder.appName("T30_Analysis").getOrCreate()

# Process a specific geographic area
result_df = process_geo_code(
    sedona=sedona,
    geo_level="LAD22CD",
    geo_code="E06000001",
    output_areas_os_tile_overlay_df=your_overlay_df,
    vom_raster_paths_df=your_vom_paths_df,
    tree_vector_paths_df=your_tree_paths_df,
    low_threshold=3,
    high_threshold=60,
    overwrite=True
)
```

### Example: Binarising Tiles

```python
from src.t30 import binarise_tiles

# Binarise CHM tiles with height thresholds
binary_chm = binarise_tiles(
    vom_paths_lst=chm_file_paths,
```

### Example: Calculating Canopy Cover

## Key Parameters

### Height Thresholds

- **Low Threshold (default: 3)**: Minimum height in meters for canopy classification
- **High Threshold (default: 60)**: Maximum height in meters for canopy classification

### Output Columns

The processed data includes:

- `OA21CD`: Output Area code
- `LSOA21CD`: Lower Layer Super Output Area code  
- `MSOA21CD`: Middle Layer Super Output Area code
- `LAD22CD`: Local Authority District code
- `RGN22CD`: Region code
- `canopy_cover`: Percentage of canopy cover (0-100)
- `total_pixels`: Total number of pixels in the analysis

## Dependencies

This module requires:

- `geopandas` for spatial data handling
- `xarray` and `rioxarray` for raster data processing
- `rasterstats` for zonal statistics
- `pyspark` for distributed computing
- `numpy` and `pandas` for data manipulation

## Notes

- The module uses the Google docstring style as configured in mkdocs.yml
- All functions include comprehensive logging for debugging and monitoring
- The binarisation process converts continuous height values to binary (0/1) based on threshold criteria
- Canopy cover is calculated as the percentage of pixels with canopy presence within each geometry
