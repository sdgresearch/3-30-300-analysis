# T3 Module Documentation

## Overview

The T3 module provides functions for counting trees within specified buffers around buildings (T3). This module is part of the 3-30-300 analysis framework for England using big spatial data, focusing on the "3 trees visible from home" component of the urban forestry rule.

## Usage Examples

### Basic Workflow

The typical workflow for T3 analysis involves:

1. **Processing VOM tiles** - Reading and filtering tree data from Vegetation Object Model files
2. **Creating building buffers** - Generating spatial buffers around buildings for tree counting
3. **Counting trees** - Calculating the number of trees within each building's buffer
4. **Processing geographic areas** - Running the complete analysis for specific geographic codes

### Example: Processing a Geographic Area

```python
from src.t3 import process_geo_code
from pyspark.sql.session import SparkSession
import geopandas as gpd
import pandas as pd

# Initialize Spark session
sedona = SparkSession.builder.appName("T3_Analysis").getOrCreate()

# Load required data
output_areas_boundaries_gdf = gpd.read_file("path/to/output_areas.gpkg")
os_tile_boundaries_gdf = gpd.read_file("path/to/os_tiles.gpkg")
output_areas_os_tile_overlay_df = pd.read_csv("path/to/overlay.csv")
vom_raster_paths_df = pd.read_csv("path/to/vom_paths.csv")
tree_vector_paths_df = pd.read_csv("path/to/tree_paths.csv")

# Process a specific geographic area
result_df = process_geo_code(
    sedona=sedona,
    query_method="sql",  # or "rdd" for Spatial RDD approach
    geo_level="LAD22CD",
    geo_code="E06000001",
    tile_level="TILE_NAME",
    output_areas_boundaries_gdf=output_areas_boundaries_gdf,
    os_tile_boundaries_gdf=os_tile_boundaries_gdf,
    output_areas_os_tile_overlay_df=output_areas_os_tile_overlay_df,
    vom_raster_paths_df=vom_raster_paths_df,
    tree_vector_paths_df=tree_vector_paths_df,
    buffer=100,  # 100 meter buffer
    tree_area=10,  # minimum tree area
    tree_height=3,  # minimum tree height
    overwrite=True
)
```

## Query Methods

### SQL Method

The SQL method uses Spark SQL to perform spatial joins and tree counting:

- **Advantages**: Familiar SQL syntax, good for smaller datasets
- **Process**: Creates temporary views and executes SQL queries
- **Use Case**: When working with moderate-sized datasets

### Spatial RDD Method

The Spatial RDD method uses Apache Sedona's spatial partitioning for distributed processing:

- **Advantages**: Better performance for large datasets, distributed processing
- **Process**: Creates spatial RDDs and uses spatial partitioning
- **Use Case**: When working with large-scale datasets

## Key Parameters

### Tree Filtering Parameters

- **tree_area (default: 10)**: Minimum tree area in square meters
- **tree_height (default: 3)**: Minimum tree height in meters
- **buffer (default: 100)**: Buffer distance around buildings in meters

### Geographic Parameters

- **geo_level**: Geographic level (e.g., "LAD22CD", "MSOA21CD")
- **geo_code**: Specific geographic code to process
- **tile_level**: Tile level for data organization
- **overwrite**: Whether to overwrite existing output files

### Query Method

- **query_method**: Either "sql" or "rdd" for different processing approaches

## Output Columns

The processed data includes:

- `verisk_premise_id`: Unique building identifier
- `tree_count`: Number of trees within the building's buffer (SQL method)
- `tree_count_{buffer}m`: Number of trees within the building's buffer (RDD method)

## Data Processing Steps

### 1. Tree Data Processing

- **VOM Tile Reading**: Reads tree data from Vegetation Object Model geopackages
- **Tree Filtering**: Filters trees based on minimum area and height criteria
- **Geometry Conversion**: Converts tree polygons to centroids for point-based analysis

### 2. Building Buffer Creation

- **Spatial Buffering**: Creates circular buffers around building centroids
- **Buffer Size**: Configurable buffer distance (typically 100 meters)
- **Spatial Indexing**: Optimizes spatial operations for performance

### 3. Tree Counting

- **Spatial Intersection**: Counts trees that intersect with building buffers
- **Aggregation**: Groups results by building ID
- **Output Generation**: Creates CSV files with tree counts per building

## Dependencies

This module requires:

- `geopandas` for spatial data handling
- `pyspark` for distributed computing
- `pandas` for data manipulation
- Apache Sedona for spatial RDD operations
- `shapely` for geometric operations

## Performance Considerations

- **Memory Usage**: Large tree datasets can consume significant memory
- **Processing Time**: Spatial operations are computationally intensive
- **Optimization**: Uses spatial indexing and partitioning for better performance
- **Scalability**: RDD method scales better for large datasets

## Notes

- The module supports two different query methods (SQL and RDD) for flexibility
- Tree filtering ensures only significant trees are counted (minimum area and height)
- Building buffers represent the "visible from home" area for tree counting
- All spatial operations use the project's coordinate reference system
- The module includes comprehensive error handling and logging
- Output files are named with geographic code and buffer distance for clarity
